# Journal index

Navigate the journal by **initiative** (a thread of related bundles) rather than
by date. Add a row per initiative; link the entries that belong to it.

| Initiative                    | Status      | Entries    |
| ----------------------------- | ----------- | ---------- |
| _(example)_ Auth & onboarding | in progress | _none yet_ |

## How to use this index

- One row per **initiative**, not per entry — an initiative spans many dated
  entries over time.
- **Status:** `in progress` / `shipped` / `paused`.
- **Entries:** link the dated files (or `…-digest.md#slug` anchors once a month
  is condensed) that make up the initiative.

See [README.md](README.md) for the entry format and the monthly-digest convention.
